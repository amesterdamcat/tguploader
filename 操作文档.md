# Telegram 视频上传器 - 操作文档

## 文件结构

```
pythonTguploader/
├── enhanced_uploader.py          # 唯一的主程序
├── .env                          # 共享设置（删除/标记/豁免文件夹等）
├── .account_configs/
│   ├── accounts.json             # 账号名称 → 手机号映射
│   ├── 855315454854.json         # wang 的凭据
│   ├── 244954186635.json         # aoo 的凭据
│   ├── 447470911238.json         # pussy 的凭据
│   ├── 85517358454.json          # ajx 的凭据
│   └── 855315376950.json         # jane 的凭据
└── session_*.session             # 每个账号独立的登录会话文件
```

## 日常操作

### 1. 查看所有账号状态

```bash
python enhanced_uploader.py accounts
```

输出示例：
```
Name       Phone              Channel              Session
--------------------------------------------------------------------
wang       +855315454854      ChaturbateRecord     OK
aoo        +244954186635      ChaturbateRecord     OK
pussy      +447470911238      ChaturbateRecord     OK
ajx        +85517358454       ChaturbateRecord     OK
jane       +855315376950      ChaturbateRecord     OK
```

Session 列显示 OK 表示有登录会话，MISSING 表示需要重新登录。

### 2. 单账号上传

```bash
# 使用默认设置上传（目录读取 .env 中的 DEFAULT_UPLOAD_DIR）
python enhanced_uploader.py upload -a wang

# 指定目录
python enhanced_uploader.py upload -a wang --dir /mnt/storage/ctbrec/media

# 不递归子目录
python enhanced_uploader.py upload -a wang --no-recursive
```

### 3. 五个账号同时上传（后台运行）

```bash
python enhanced_uploader.py upload -a wang &
python enhanced_uploader.py upload -a aoo &
python enhanced_uploader.py upload -a pussy &
python enhanced_uploader.py upload -a ajx &
python enhanced_uploader.py upload -a jane &
```

每个账号读自己独立的配置，互不影响。文件夹锁机制防止两个账号同时上传同一个文件夹。

### 4. 调整上传速度参数

```bash
python enhanced_uploader.py upload -a wang \
    --batch-size 200 \
    --concurrent 4 \
    --delay 0.5
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--batch-size` | 100 | 每批处理多少个块 |
| `--concurrent` | 1 | 每批内同时上传几个任务 |
| `--delay` | 1.0 | 每个任务之间延迟（秒） |

**推荐组合：**

| 场景 | 命令 |
|------|------|
| 稳定上传 | `--batch-size 100 --concurrent 1 --delay 1.0` |
| 加速上传 | `--batch-size 200 --concurrent 4 --delay 0.5` |
| 极速上传（可能限流） | `--batch-size 500 --concurrent 8 --delay 0.1` |

### 5. 账号过期重新登录

```bash
python enhanced_uploader.py login -a wang
```

会删除旧的 session 文件，然后进入 Telegram 登录流程（输入验证码）。

## 配置修改

### 修改共享设置（.env）

直接编辑 `.env` 文件：

```env
DELETE_AFTER_UPLOAD=true          # 上传后是否删除源文件
MARK_UPLOADED_FILES=true          # 是否给已上传文件加后缀
UPLOADED_SUFFIX=.uploaded         # 后缀名
EXEMPT_FOLDERS=/path/a,/path/b   # 豁免删除的文件夹（逗号分隔）
DEFAULT_UPLOAD_DIR=/mnt/storage/ctbrec/media  # 默认上传目录
```

### 修改账号名称映射

编辑 `.account_configs/accounts.json`：

```json
{
  "wang": "855315454854",
  "aoo": "244954186635",
  "pussy": "447470911238",
  "ajx": "85517358454",
  "jane": "855315376950"
}
```

左边是你用 `--account` 指定的名字，右边是手机号（不带+号）。

### 修改账号凭据（API_ID 等）

编辑 `.account_configs/{手机号}.json`：

```json
{
  "phone": "+855315454854",
  "api_id": "20358207",
  "api_hash": "b931d99e26b92c26acc77469daecede2",
  "channel_id": "ChaturbateRecord"
}
```

### 添加新账号

1. 在 `.account_configs/` 下创建 `{手机号}.json`，填入凭据
2. 在 `accounts.json` 中添加名称映射
3. 运行 `python enhanced_uploader.py login -a 新名字` 登录

## 故障排查

| 问题 | 解决 |
|------|------|
| `Session: MISSING` | 运行 `python enhanced_uploader.py login -a 账号名` |
| `Error: account 'xxx' not found` | 检查 `accounts.json` 中是否有这个名字 |
| `Error: directory not found` | 检查 `--dir` 路径或 `.env` 中的 `DEFAULT_UPLOAD_DIR` |
| `Error: channel not found` | 检查账号 JSON 中的 `channel_id` 是否正确 |
| `FLOOD_WAIT` 限流 | 增大 `--delay`，减小 `--concurrent` |
| 所有文件夹都被锁定 | 检查是否有僵死的上传进程，或等待锁自动超时（6小时） |
| 文件被标记为 `.big` | 文件超过 2GB，Telegram 免费账号无法上传 |
